# Neatoo 0.22.0

**Release Date:** 2026-03-14
**Type:** Feature
**Breaking Changes:** Yes — LazyLoad properties must be `partial`; `SubscribeToLazyLoadProperties()` removed

---

## Summary

LazyLoad properties are now fully unified into PropertyManager via source-generated look-through property subclasses. Consumers declare `public partial LazyLoad<T> Prop { get; set; }` — the generator handles backing fields, registration, and meta property propagation. All runtime reflection-based LazyLoad discovery and the parallel property system in ValidateBase are eliminated.

---

## What's New

### LazyLoad Properties Are Partial

LazyLoad properties follow the same pattern as every other Neatoo property — declare as `partial` and the generator handles everything:

```csharp
// Before (0.21.0) — manual backing field and ceremony
private LazyLoad<IChild> _lazyChild = null!;
public LazyLoad<IChild> LazyChild
{
    get => _lazyChild;
    private set
    {
        _lazyChild = value;
        SubscribeToLazyLoadProperties();
    }
}

// After (0.22.0) — partial, just like every other property
public partial LazyLoad<IChild> LazyChild { get; set; }
```

The generator produces a `LazyLoadValidateProperty<T>` (or `LazyLoadEntityProperty<T>` for EntityBase) backing field that sees through to the inner entity for all framework operations.

### Look-Through Property Subclasses

`LazyLoadValidateProperty<T>` and `LazyLoadEntityProperty<T>` are PropertyManager-native subclasses that look through the `LazyLoad<T>` wrapper to its `.Value`. Framework code outside these subclasses is unaware of the LazyLoad nature of the property:

- `IValidateProperty.Value` returns the inner entity, not the wrapper
- `SetParent` is called on the inner entity when it loads
- `IsBusy`, `IsValid`, `IsModified` propagate from the inner entity
- `PropertyMessages` aggregate from the inner entity
- RunRules cascading works through the inner entity

### LazyLoad.SetValue(T?)

New method for direct value assignment, bypassing the loader delegate. Marks the instance as loaded, clears any load error, and manages child event subscriptions:

```csharp
[Create]
public void Create([Service] IPersonPhoneList emptyPhoneList)
{
    PersonPhoneList.SetValue(emptyPhoneList);
}
```

### Value Propagation

The PropertyManager event structure propagates `.Value` (the inner entity), not the `LazyLoad<T>` wrapper. When the inner entity is not loaded, it appears as `null` to the framework. When loaded, it behaves identically to setting a property value directly.

### Generator Detects LazyLoad Types

`PropertyExtractor` detects `LazyLoad<T>` via `IPropertySymbol.Type.OriginalDefinition` and sets `IsLazyLoad` and `LazyLoadInnerType` on `PartialPropertyInfo`. `InitializerGenerator` emits `factory.CreateLazyLoad<TInner>()` instead of `factory.Create<Type>()`.

### NEATOO010 Analyzer Updated

The `ConstructorPropertyAssignmentAnalyzer` now skips LazyLoad properties — their generated setter uses `LoadValue`, so constructor assignment is safe.

## Breaking Changes

### LazyLoad Properties Must Be `partial`

Manual backing field declarations with `SubscribeToLazyLoadProperties()` no longer work. Convert to partial properties:

```csharp
// Remove this:
private LazyLoad<IChild> _child = null!;
public LazyLoad<IChild> Child
{
    get => _child;
    private set { _child = value; SubscribeToLazyLoadProperties(); }
}

// Replace with:
public partial LazyLoad<IChild> Child { get; set; }
```

### Removed APIs

| Removed | Replacement |
|---------|-------------|
| `ValidateBase.SubscribeToLazyLoadProperties()` | Automatic via generator |
| `ValidateBase.RegisterLazyLoadProperty<T>()` | Automatic via generator |
| `ValidateBase.RegisterLazyLoadProperties()` | Automatic via generator |
| `ValidatePropertyManager.FinalizeRegistration()` | Automatic via generator |
| `ValidateBase` LazyLoad state aggregation (`IsAnyLazyLoadChildBusy`, `IsAllLazyLoadChildrenValid`, etc.) | PropertyManager handles via look-through subclasses |
| `EntityBase.IsAnyLazyLoadChildModified()` | PropertyManager handles via `LazyLoadEntityProperty<T>` |

### IPropertyFactory.CreateLazyLoad<TInner>

New method on `IPropertyFactory` — implementations must provide `CreateLazyLoad<TInner>()`. `DefaultPropertyFactory` creates `LazyLoadValidateProperty<TInner>`, `EntityPropertyFactory` creates `LazyLoadEntityProperty<TInner>`.

---

## Migration

1. Convert all LazyLoad properties to `partial`:
   ```csharp
   public partial LazyLoad<IChild> Child { get; set; }
   ```
2. Remove manual backing fields, custom setters, and `SubscribeToLazyLoadProperties()` calls
3. Use `LazyLoad<T>.SetValue(T?)` in `[Create]` methods instead of assigning a pre-loaded instance (or assign directly — the generated setter uses `LoadValue`)
4. Constructor still creates `LazyLoad<T>` with the loader delegate via `ILazyLoadFactory` — this hasn't changed

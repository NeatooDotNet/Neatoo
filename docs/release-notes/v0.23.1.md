# Neatoo 0.23.1

**Release Date:** 2026-03-21
**Type:** Patch
**Breaking Changes:** No

---

## Summary

Updates RemoteFactory dependency from 0.21.0 to 0.23.1, picking up shared reference handling for mutable types and a factory fetch NRE fix.

---

## What's New

### RemoteFactory 0.21.0 → 0.23.1

**Shared reference identity for mutable types (0.23.0).** When two properties reference the same `Dictionary`, `List`, or plain class instance, RemoteFactory now preserves that identity across serialization round-trips. Previously, shared references became independent copies after serialization — `ReferenceEquals` returned `false`. Now both properties point to the same object after deserialization.

Records and other parameterized-constructor types (value objects) correctly bypass reference tracking — duplication is correct DDD behavior for value objects.

**Factory fetch NRE fix (0.23.1).** Fixed a `NullReferenceException` when a `[Fetch]` method returns `false` (not found) on a factory decorated with `[AuthorizeFactory]`.

# Neatoo 0.15.1

**Release Date:** 2026-03-03
**Type:** Bug Fix
**Breaking Changes:** No

---

## Summary

Fixes `IsValid`/`IsSelfValid`/`IsSavable` returning stale values when `RunRules()` is called during a factory operation (e.g., inside `[Insert]` or `[Update]` methods). `[Required]` and other data annotation rules now correctly affect validation state during save.

---

## Bug Fixes

### IsValid Stale During RunRules() in Factory Operations

When calling `RunRules()` inside a factory method like `[Insert]`, validation rules executed correctly and set error messages on properties, but `IsValid` remained `true`. This caused `IsSavable` to incorrectly return `true`, allowing saves to proceed with invalid data.

**Affected pattern:**

```csharp
[Insert]
public async Task Insert([Service] IRepository repository)
{
    await RunRules();

    if (!this.IsSavable)  // Always true — bug
        return;

    // Save proceeded with invalid data
}
```

**Root cause:** `ValidatePropertyManager.IsValid` is a cached property that recalculates via `PropertyChanged` events. Factory operations pause the property manager (`FactoryStart` calls `PauseAllActions`), which suppresses events. Rules set error messages on properties, but the cached `IsValid` was never updated.

**Fix:** After `RunRules()` completes, the cached `IsValid`/`IsSelfValid` values are now recalculated from actual property state, regardless of paused state. This uses the same recalculation logic already present in `ResumeAllActions()`.

---

## Migration

No code changes required. This is a pure bug fix with no API changes.

If you have workarounds that avoid checking `IsSavable` or `IsValid` inside factory methods, those workarounds can now be removed.

# Neatoo 0.15.0

**Release Date:** 2026-03-02
**Type:** Breaking Change
**Breaking Changes:** Yes

---

## Summary

Introduces `IEntityRoot` interface to separate aggregate root operations (`IsSavable`, `Save()`) from child entity operations. This prevents a production bug pattern where `IsSavable` on child entities silently returned `false`, causing save cascades to skip persistence.

---

## What's New

### IEntityRoot interface

A new `IEntityRoot` interface extends `IEntityBase` with `IsSavable` and `Save()`. Aggregate root entity interfaces should extend `IEntityRoot`; child entity interfaces should extend `IEntityBase`.

```csharp
// Aggregate root -- exposes IsSavable and Save()
public interface IOrder : IEntityRoot { ... }

// Child entity -- no IsSavable, no Save()
public interface IOrderLine : IEntityBase { ... }
```

### IsSavable removed from IEntityBase, IEntityMetaProperties, and EntityListBase

`IsSavable` now lives exclusively on `IEntityRoot`. It was removed from:
- `IEntityBase` -- child entities should never expose `IsSavable` (it was always false due to the `!IsChild` check)
- `IEntityMetaProperties` -- prevents entity lists and child entities from surfacing a property that was always false
- `EntityListBase` -- lists are always saved through the aggregate root

### Save() removed from IEntityBase

`Save()` now lives exclusively on `IEntityRoot`. Child entities cannot initiate their own save -- this is enforced at the type level rather than with a runtime exception.

---

## Breaking Changes

### Interface changes

| Interface | Removed Members | Replacement |
|-----------|----------------|-------------|
| `IEntityBase` | `IsSavable`, `Save()`, `Save(CancellationToken)` | Use `IEntityRoot` for aggregate roots |
| `IEntityMetaProperties` | `IsSavable` | Use `IEntityRoot` for aggregate roots |
| `EntityListBase` | `IsSavable` property | N/A (was always false) |

### Migration Guide

1. **Aggregate root interfaces**: Change from `IEntityBase` to `IEntityRoot`:
   ```csharp
   // Before
   public interface IOrder : IEntityBase { }

   // After
   public interface IOrder : IEntityRoot { }
   ```

2. **Child entity interfaces**: Keep using `IEntityBase` (no change needed):
   ```csharp
   public interface IOrderLine : IEntityBase { }
   ```

3. **Code accessing `IsSavable` on child entities**: This was always returning `false`. Remove the check or use `IsModified` / `IsNew` as appropriate for your logic.

4. **Code accessing `IsSavable` via `IEntityMetaProperties`**: Cast to `IEntityRoot` instead:
   ```csharp
   // Before
   IEntityMetaProperties meta = ...;
   if (meta.IsSavable) { ... }

   // After
   if (meta is IEntityRoot root && root.IsSavable) { ... }
   ```

# Neatoo 0.19.0

**Release Date:** 2026-03-08
**Type:** Dependency
**Breaking Changes:** No

---

## Summary

Updates RemoteFactory to 0.21.0 and makes all `[Remote]` factory methods `internal` to comply with the new NF0105 analyzer rule. This enforces IL trimming correctness by ensuring server-only code is not publicly accessible from client assemblies.

---

## What's New

### RemoteFactory 0.21.0

Both `Neatoo.RemoteFactory` and `Neatoo.RemoteFactory.AspNetCore` updated from 0.20.1 to 0.21.0.

### NF0105: [Remote] Methods Must Be Internal

RemoteFactory 0.21.0 introduces analyzer rule NF0105 which requires all `[Remote]` methods to have `internal` accessibility. This prevents client assemblies from referencing server-only code, enabling the IL trimmer to eliminate it entirely from Blazor WASM bundles.

All `[Remote]` methods in the Person example, code samples, and documentation snippets have been updated from `public` to `internal`.

---

## Migration

Change all `[Remote]` factory methods from `public` to `internal`:

```csharp
// Before
[Remote]
[Fetch]
public async Task Fetch(int id, [Service] IRepository repo) { ... }

// After
[Remote]
[Fetch]
internal async Task Fetch(int id, [Service] IRepository repo) { ... }
```

No other code changes required. The generated factory interfaces remain `public` and are unaffected.

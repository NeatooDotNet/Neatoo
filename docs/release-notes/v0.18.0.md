# Neatoo 0.18.0

**Release Date:** 2026-03-07
**Type:** Feature
**Breaking Changes:** No

---

## Summary

Adds IL trimming support for Blazor WASM consumers. Consumer domain model projects can enable `<IsTrimmable>true</IsTrimmable>` and Blazor WASM apps can publish with `<PublishTrimmed>true</PublishTrimmed>` without Neatoo causing trimming warnings. Server-only code (internal factory methods, repository dependencies) is eliminated from the published output.

---

## What's New

### IL Trimming Annotations

Neatoo's core type parameters are now annotated with `[DynamicallyAccessedMembers]` to inform the IL trimmer which members to preserve. This covers the property discovery system (`PropertyInfoList<T>`), service cascade (`IValidateBaseServices<T>`, `IEntityBaseServices<T>`, etc.), and rule management (`IRuleManager<T>`).

Reflection-heavy internal methods (JSON converters, `RequiredRule`, `ValidatePropertyManager`) have `[UnconditionalSuppressMessage]` attributes with documented justifications explaining why each suppression is safe.

### EnableTrimAnalyzer

`Neatoo.csproj` now has `<EnableTrimAnalyzer>true</EnableTrimAnalyzer>` for ongoing detection of new trim-unsafe patterns. Neatoo itself is NOT marked `<IsTrimmable>` -- only consumer projects opt in.

### Consumer Project Trimming

Consumer domain model projects can add `<IsTrimmable>true</IsTrimmable>`. Blazor WASM apps can publish with:

```xml
<PropertyGroup>
    <PublishTrimmed>true</PublishTrimmed>
    <TrimMode>full</TrimMode>
</PropertyGroup>

<ItemGroup>
    <RuntimeHostConfigurationOption Include="NeatooRuntime.IsServerRuntime"
                                    Value="false"
                                    Trim="true" />
</ItemGroup>
```

### Verified Size Reductions (Person Example)

| Assembly | Untrimmed | Trimmed | Reduction |
|----------|-----------|---------|-----------|
| Person.DomainModel | 59,157 bytes | 4,373 bytes | 93% |
| Neatoo | 143,637 bytes | 60,693 bytes | 58% |
| Person.Dal | 6,421 bytes | Removed entirely | 100% |
| Neatoo.Blazor.MudNeatoo | 63,765 bytes | Removed entirely | 100% |

---

## Migration

No code changes required. This is additive -- `[DynamicallyAccessedMembers]` on type parameters is metadata that does not alter runtime behavior. All existing code continues to compile and run identically.

To enable trimming in your projects, follow the consumer project configuration in the IL Trimming documentation.

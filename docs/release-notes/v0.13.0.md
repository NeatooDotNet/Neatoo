# Neatoo 0.13.0

**Release Date:** 2026-03-01
**Type:** Bug Fix
**Breaking Changes:** No

---

## Summary

Fixes stale meta property caches (`IsModified`, `IsValid`, `IsSelfModified`, `IsSelfValid`, `IsBusy`) after `ResumeAllActions()` in `EntityPropertyManager` and `ValidatePropertyManager`. Parent entities now correctly reflect child state changes that occur during factory operations.

---

## Bug Fixes

### Meta Property Caches Recalculated After Resume

When a parent entity was paused during `[Create]` or `[Fetch]`, child entities going through independent factory lifecycles could become modified or invalid. The parent's property manager received `PropertyChanged` events but dropped them because it was paused. On resume, cached values were never recalculated — the parent reported `IsModified=false` while children were genuinely modified.

**Root cause (two issues):**

1. **Missing recalculation on resume.** Neither `EntityPropertyManager.ResumeAllActions()` nor `ValidatePropertyManager.ResumeAllActions()` recalculated cached meta properties from current property state. Events received while paused were silently dropped.

2. **Broken inheritance.** `EntityPropertyManager` declared `PauseAllActions()`, `ResumeAllActions()`, and `IsPaused` with implicit `new` instead of `override`, hiding `ValidatePropertyManager`'s virtual methods. Due to C# interface re-implementation, `ValidatePropertyManager.IsPaused` was never set to `true` for `EntityBase` objects.

**Fix:**

- Refactored `EntityPropertyManager` to use `override` instead of implicit `new` for `PauseAllActions()` and `ResumeAllActions()`
- Removed `EntityPropertyManager`'s hidden `IsPaused` property
- Added `IsModified`/`IsSelfModified` recalculation to `EntityPropertyManager.ResumeAllActions()`
- Added `IsValid`/`IsSelfValid`/`IsBusy` recalculation to `ValidatePropertyManager.ResumeAllActions()`

### VPM Event Processing During EPM Pause (Secondary Fix)

`ValidatePropertyManager.Property_PropertyChanged` was processing events (recalculating `IsValid`/`IsSelfValid`) during `EntityPropertyManager` pause because `VPM.IsPaused` was never set. This unnecessary work is now correctly suppressed, with recalculation deferred to resume.

---

## Migration

No code changes required. This is a pure bug fix with no API changes.

If you have workarounds using explicit `MarkModified()` calls after factory operations to compensate for stale `IsModified`, those calls can now be removed.

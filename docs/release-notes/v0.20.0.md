# Neatoo 0.20.0

**Release Date:** 2026-03-12
**Type:** Enhancement
**Breaking Changes:** No

---

## Summary

`LazyLoad<T>` now supports nullable reference types. The generic constraint was relaxed from `where T : class` to `where T : class?`, allowing declarations like `LazyLoad<IOrderItemList?>`.

---

## What's New

### Nullable Reference Type Support for LazyLoad

The `where T : class` constraint on `LazyLoad<T>`, `ILazyLoadFactory`, and `LazyLoadFactory` has been changed to `where T : class?`. This allows using nullable reference types as the type parameter:

```csharp
// Now valid — previously a compiler error
public LazyLoad<IOrderItemList?> OrderItems { get; set; }
```

Existing code using non-nullable types (e.g., `LazyLoad<IOrderItemList>`) is unaffected.

# Neatoo 0.14.1

**Release Date:** 2026-03-02
**Type:** Bug Fix
**Breaking Changes:** No

---

## Summary

Fixes Blazor WASM publish failures caused by `Microsoft.Extensions.DependencyInjection` version mismatch. The net10.0 central package version (10.0.3) was bleeding into net9.0 WASM builds. Each target framework now uses an explicit `VersionOverride` to pin the correct DI version.

---

## What's Fixed

- Added explicit `VersionOverride="10.0.3"` for `Microsoft.Extensions.DependencyInjection` on the net10.0 target in `Neatoo.csproj`
- net8.0 → 8.0.1, net9.0 → 9.0.5, net10.0 → 10.0.3

# Neatoo 0.23.2

**Release Date:** 2026-03-21
**Type:** Patch
**Breaking Changes:** No

---

## Summary

Migrates Neatoo's custom JSON converters to RemoteFactory's `NeatooReferenceResolver.Current` API. Both Neatoo's custom converters and STJ's built-in converters now share the same reference resolver instance, enabling cross-type `$id`/`$ref` tracking during serialization.

---

## What Changed

### Converter Migration to NeatooReferenceResolver.Current

`NeatooBaseJsonTypeConverter` and `NeatooListBaseJsonTypeConverter` no longer call `options.ReferenceHandler.CreateResolver()`. Instead, they use `NeatooReferenceResolver.Current` directly — the same resolver instance that RemoteFactory's `NeatooPreserveReferenceHandler` bridges to for STJ's built-in converters.

This is an internal implementation change with no public API impact.
